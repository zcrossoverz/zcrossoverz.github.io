1. Bài Toán Thực Tế — 6 Lỗ Hổng Lộ Ra Trong 30 Giây

Tình huống: Junior 1 năm kinh nghiệm phỏng vấn Grab, fail vòng coding

Đề bài: "Cho list of transactions, group theo userId rồi return userId nào có tổng amount > 1tr."

Code junior viết — chạy được, nhưng interviewer "kill" ngay
public List<String> findTopUsers(List<Transaction> txns) {
    List<String> result = new ArrayList<>();
    HashMap<String, Long> sums = new HashMap<>();

    for (int i = 0; i < txns.size(); i++) {
        Transaction t = txns.get(i);
        if (sums.containsKey(t.getUserId())) {
            sums.put(t.getUserId(), sums.get(t.getUserId()) + t.getAmount());
        } else {
            sums.put(t.getUserId(), t.getAmount());
        }
    }

    for (String userId : sums.keySet()) {
        if (sums.get(userId) > 1000000) {
            result.add(userId);
        }
    }
    return result;
}

Code chạy đúng output. Nhưng senior interviewer ngồi đối diện sẽ nghĩ những điều sau, trong vòng 30 giây đầu nhìn code:

Điểm yếu trong code Senior nhận xét
for (int i = 0; ...; i++) rồi txns.get(i) "Sao không dùng for-each? Nếu list là LinkedList thì .get(i) là O(n²) đó."
containsKey + get + put (3 lookups) "1 lookup là đủ với merge() hoặc getOrDefault(). Junior thiếu fluency với Map API."
for (String userId : sums.keySet()) rồi sums.get(userId) "Lại 2 lookups. entrySet() là chuẩn. Java basics chưa solid."
Trả ArrayList không sized "Đoán capacity được không? Nếu không, JVM sẽ resize nhiều lần."
Dùng HashMap<String, Long> cho amount "Tiền tệ dùng Long mất chữ thập phân, dùng BigDecimal chứ. Banking domain chưa nghiêm túc."
⚠️ Đây là gap "senior thấy ngay trong 30 giây"

Không cần biết DP/Backtracking, chỉ cần fluent với Java Collections đã khác biệt 1 level. Đây là lý do junior bị reject ở vòng phỏng vấn coding mà không hiểu vì sao — "code chạy đúng mà".

5 câu hỏi tự test trước khi đọc tiếp

  1. ArrayList.get(i) là O(?) — LinkedList.get(i) là O(?). Vậy khi nào nên dùng LinkedList?
  2. HashMap.put(k, v) average O(1) — worst case là O(?). Java 8 fix worst case xuống còn O(?).
  3. Integer a = 127; Integer b = 127; a == b → true hay false? Còn a = 128; b = 128?
  4. try { return 1; } finally { return 2; } — return giá trị nào?
  5. Recursion của tree height — base case là gì? Vì sao base case quan trọng hơn recursive case?

Nếu trả lời chậm hoặc không chắc bất kỳ câu nào → file này dành cho bạn. Đọc tiếp.

2. Mental Model — 6 Lớp Nền

Lớp 1: Collections — Bản Đồ Khi Nào Dùng Cái Nào

Đây là kiến thức bạn dùng hàng ngày. Chọn sai = bug performance ẩn (chạy ngon dev, chậm chết prod khi data lớn). Vẽ ra cây quyết định trong đầu để chọn nhanh:

Cây quyết định chọn collection theo nhu cầu nghiệp vụ

Bảng tham chiếu nhanh — học thuộc

Tình huống Chọn Lý do
List + index access nhanh + biết size ArrayList(initialCapacity) O(1) random access, contiguous memory, cache locality tốt
List + add/remove ở đầu/cuối ArrayDeque O(1) cả 2 đầu, không có overhead Node của LinkedList
List + add/remove giữa LinkedList (HIẾM) Chỉ khi đã có Iterator ở vị trí cần insert/remove
Stack ArrayDeque (KHÔNG Stack) Stack extends Vector, synchronized, legacy slow
Queue ArrayDeque hoặc LinkedList LinkedList implements Queue interface
FIFO + thread-safe LinkedBlockingQueue Producer-consumer pattern
Priority queue (min-heap default) PriorityQueue O(log n) offer/poll, root là min
Set + fast contains HashSet O(1) average
Set + sorted iteration TreeSet O(log n), Red-Black tree
Set + insertion order LinkedHashSet O(1) + giữ thứ tự thêm
Map + fast HashMap O(1) average
Map + sorted by key TreeMap O(log n), range queries
Map + insertion/access order LinkedHashMap LRU cache built-in
Map + thread-safe ConcurrentHashMap Lock-free reads, sharded writes

Anti-pattern phổ biến nhất

❌ Junior — học theo sách cũ
// Stack legacy
Stack<Integer> stack = new Stack<>();
// extends Vector → synchronized → slow

// Arrays.asList rồi add → BÙM
List<String> list =
    Arrays.asList("a", "b", "c");
list.add("d");
// UnsupportedOperationException!
// (fixed-size list)
✅ Senior
// Deque cho stack
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.pop();
stack.peek();

// Mutable list đúng cách
List<String> list = new ArrayList<>(
    Arrays.asList("a", "b", "c"));
// Java 9+:
// new ArrayList<>(List.of("a","b","c"))

Lớp 2: Generics — Đọc Được PECS

Khi đọc Spring/Java standard library, bạn sẽ gặp hàng trăm signature như:

public static <T> void copy(List<? super T> dest, List<? extends T> src);
public static <T extends Comparable<? super T>> void sort(List<T> list);

Không hiểu PECS = không đọc được Java standard library. PECS đứng cho Producer Extends, Consumer Super.

🧠 Mental model PECS

Producer = source bạn đọc dữ liệu ra (như Iterator) → dùng extends
Consumer = sink bạn ghi dữ liệu vào (như Collector) → dùng super

Producer — đọc từ collection, dùng extends
public double sumOfList(List<? extends Number> list) {
    double sum = 0;
    for (Number n : list) sum += n.doubleValue();  // ✅ Đọc OK
    // list.add(...);  ❌ Compile error: không biết list là List hay List
    return sum;
}

// Caller có thể truyền bất kỳ List với X là subclass Number
sumOfList(List.of(1, 2, 3));            // List
sumOfList(List.of(1.5, 2.5));           // List
sumOfList(List.of(new BigDecimal("1"))); // List
Consumer — ghi vào collection, dùng super
public void addIntegers(List<? super Integer> list) { list.add(1); // ✅ OK: list có thể là List Integer, List Number, List Object list.add(2); // Integer x = list.get(0); ❌ chỉ get được Object } // Caller truyền List, List, hoặc List
