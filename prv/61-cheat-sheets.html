#01 Java Concurrency

Visibility + atomicity + đừng tin synchronized trên multi-instance.

  • Thread pool: core đầy → queue → queue đầy → tới max → reject. newFixedThreadPool = queue unbounded → OOM.
  • volatile = visibility; synchronized = visibility + mutual exclusion; Atomic = CAS, nhanh khi ít tranh chấp.
  • Chống double-pay: idempotency key + DB lock → distributed lock → Saga.

  • Phân biệt FAILED transient (retry) vs business (non-retry). Key do client sinh.
